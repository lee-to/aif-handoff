# /aif-review — Code review

Проводит детальный code review: ищет баги, проблемы безопасности, нарушения архитектуры и best practices.

## Использование

```
/aif-review              # review staged изменений
/aif-review PR#42        # review конкретного PR
/aif-review main..feat   # review диапазона коммитов
```

## Что проверяет

- **Корректность** — логические ошибки, edge cases, null/undefined
- **Безопасность** — инъекции, XSS, открытые данные
- **Производительность** — N+1 запросы, лишние вычисления
- **Архитектура** — соответствие ARCHITECTURE.md, layer boundaries
- **Правила** — соответствие RULES.md и соглашениям проекта
- **Тесты** — покрытие изменений тестами

## Результат

Структурированный отчёт с находками по severity:

- `ERROR` — нужно исправить до merge
- `WARN` — желательно исправить
- `INFO` — рекомендация / улучшение

## После review

```
/aif-fix   # исправить найденные баги
/aif-commit   # если замечаний нет
```

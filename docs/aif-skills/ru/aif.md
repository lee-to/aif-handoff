# /aif — Инициализация проекта

Настраивает AI Factory в проекте с нуля: анализирует tech stack, устанавливает релевантные скиллы, генерирует кастомные скиллы, конфигурирует MCP серверы.

**Запускай один раз при старте проекта или после смены tech stack.**

## Использование

```
/aif
```

## Что делает

1. Читает `.ai-factory/DESCRIPTION.md` — узнаёт tech stack проекта
2. Устанавливает подходящие скиллы из реестра (`skills.sh`)
3. Генерирует кастомные скиллы если нужны
4. Конфигурирует MCP серверы
5. Инициализирует структуру `.ai-factory/`

## Структура которую создаёт

```
.ai-factory/
├── DESCRIPTION.md      # описание проекта и tech stack
├── ARCHITECTURE.md     # архитектурные решения
├── RULES.md            # правила и соглашения
├── PLAN.md             # текущий план
├── patches/            # патчи исправлений (для aif-evolve)
├── skill-context/      # проектные переопределения скиллов
└── references/         # knowledge references
```

## Когда использовать

- Новый проект
- Подключение к существующему проекту первый раз
- После существенного изменения tech stack

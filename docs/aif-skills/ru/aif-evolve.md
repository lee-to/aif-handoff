# /aif-evolve — Самообучение скиллов

Анализирует накопленные патчи исправлений и улучшает скиллы, добавляя проектно-специфичные правила. Это механизм "памяти" AI Factory.

## Использование

```
/aif-evolve              # обновить все скиллы
/aif-evolve aif-fix      # обновить конкретный скилл
/aif-evolve aif-commit aif-implement   # несколько скиллов
```

## Как работает

```
patches/*.md (прошлые исправления)
    ↓
/aif-evolve анализирует паттерны ошибок
    ↓
выявляет prevention points
    ↓
записывает правила в .ai-factory/skill-context/<skill>/SKILL.md
    ↓
скиллы читают skill-context при каждом запуске
    ↓
ошибки больше не повторяются
```

## Patch-файлы

Патчи создаются автоматически через `/aif-fix`. Хранятся в `.ai-factory/patches/YYYY-MM-DD.md`. Каждый патч содержит:

- Описание проблемы
- Root cause
- Prevention points (конкретные действия для предотвращения)

## Skill-context

Результат работы `/aif-evolve` — файлы в `.ai-factory/skill-context/<skill-name>/SKILL.md`.

Эти файлы:

- **Не перезаписываются** при обновлении скиллов
- **Имеют приоритет** над базовыми SKILL.md при конфликте
- **Читаются обязательно** каждым скиллом перед выполнением

## Инкрементальность

Используется cursor-based подход — каждый запуск обрабатывает только новые патчи. Курсор хранится в `.ai-factory/evolutions/patch-cursor.json`.

## Когда запускать

- После нескольких исправлений через `/aif-fix`
- Периодически (раз в неделю при активной разработке)
- Когда замечаешь что одни и те же ошибки повторяются

## Важно

- Никогда не редактирует базовые файлы в `.claude/skills/aif-*/`
- Все изменения только в `.ai-factory/skill-context/`
- Логи эволюции сохраняются в `.ai-factory/evolutions/`

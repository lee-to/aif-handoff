# /aif-improve — Улучшение плана

Вторая итерация планирования: переанализирует кодовую базу, находит пробелы и улучшает существующий план.

## Использование

```
/aif-improve
```

Запускай после `/aif-plan` если план кажется неполным.

## Что делает

1. Загружает существующий `PLAN.md`
2. Переанализирует кодовую базу с повышенной тщательностью
3. Проверяет пробелы, неправильные зависимости между задачами
4. Проверяет что учтены все edge cases
5. Обновляет план с улучшениями

## Когда использовать

- После `/aif-plan` перед началом реализации
- Если в процессе реализации обнаружились незапланированные задачи
- Если план кажется слишком поверхностным

## Типичный workflow

```
/aif-plan новая фича
/aif-improve          # ← опциональный шаг улучшения
/aif-implement
```

# /aif-rules — Правила проекта

Создаёт и обновляет `RULES.md` — документ с соглашениями и правилами проекта.

## Использование

```
/aif-rules
/aif-rules update   # обновить на основе текущего кода
```

## Что создаёт

Файл `.ai-factory/RULES.md` с соглашениями:

- Именование файлов, переменных, функций
- Структура модулей
- Правила коммитов
- Обработка ошибок
- Логирование
- Тестирование

## Зачем нужен RULES.md

Все скиллы читают RULES.md как контекст. Чем точнее правила — тем лучше качество генерируемого кода и коммитов.

## Пример содержимого

```markdown
## Naming

- Files: kebab-case (user-service.ts)
- Classes: PascalCase
- Constants: SCREAMING_SNAKE_CASE

## Commits

- Scope обязателен
- Тикет JIRA в footer если есть

## Error Handling

- Всегда логировать с context объектом
- Никогда не глотать ошибки в catch
```
